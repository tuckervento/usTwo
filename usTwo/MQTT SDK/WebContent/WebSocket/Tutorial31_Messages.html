Messages.

The message can be an ArrayBuffer or a string. The qos determines how reliably the message is delivered, qos=0 means the message may be lost of there is a network failure. qos=1 means that even if there is network failure, or the client stops, the message is delivered when the client restarts or reconnects. qos=2 means that the message is always delivered as with qos=1, but it is never delivered more than once.
